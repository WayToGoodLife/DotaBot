### Название

### Суть
Автоснайп «игра сам с собой» через Steam аккаунты (Dota 2).  
Управление — Telegram бот или через саму программу.  
Запуск — /start в боте или кнопка в программе.

## 1. Telegram-бот (управление)

### Подключение
- Пользователь создаёт бота у @BotFather → получает уникальный токен.
- Вставляет токен в программу один раз.
- Программа подключается к боту.

### Команды бота

| Команда | Действие |
|---------|----------|
| /start | Запуск снайпа (всё автоматически) |
| /status | Статус: сколько акков в лобби, время матча, стадия |
| /stop | Остановка снайпа, закрытие всех sandbox-окон |
| /stats | Статистика побед/поражений (система 50%) |
| /screenshot | Сделать скриншот экрана и прислать в Telegram |

### Бот присылает автоматически (уведомления)
- Матч найден
- Прошло заданное время (настраивается)
- GG отправлен → проигрыш
- Матч закончен + кто победил
- ⚠ Ошибки (не запустился sandbox, не нашёлся матч, аккаунт не залогинился, Dota 2 не открылась, окна не расставились, боты не заснайпились)
- 📸 Скриншот по команде /screenshot

## 2. Запуск снайпа

### Два способа запуска

| Способ | Как работает |
|--------|--------------|
| Через Telegram бота | Пользователь пишет /start в боте → программа запускает снайп |
| Через саму программу | В интерфейсе программы есть кнопка «Запуск» / «Start» → нажал → снайп запустился |

### Циклический режим
- После окончания матча программа сама запускает следующий.
- Работает бесконечно, пока пользователь не отправит /stop или не нажмёт кнопку остановки в программе.

---

## 3. Поиск матча

### Выбор сервера (автоматически)
- Сервер — Индия (India / Chennai).
- Программа автоматически выбирает индийский сервер.
- Пользователь не выбирает регион — это зафиксировано.

### Повторный поиск
- Если матч не нашёлся за N минут (настраивается пользователем) → сброс поиска и начало заново.
- Если боты не заснайпились (не попали в одну игру) → отмена МГР → новый поиск.

## 4. Логины / пароли (в программе)

- Таблица на 5–10 аккаунтов Steam: логин + пароль.
- Программа сама логинит каждый аккаунт.
- Steam Guard: запросить код у пользователя 1 раз → запомнить сессию.

## 5. Sandbox-окна — полная автоматизация

### Открытие
- Каждый аккаунт запускается в отдельной песочнице (sandbox).
- Аккаунты не пересекаются.

### Расстановка по экрану
- Программа сама открывает окна и располагает их сеткой (например 2×3, 3×3).
- Окна не накладываются друг на друга.
- Автоматическая расстановка по экрану монитора.

### Настройки Dota 2 (автоматически)
При первом входе каждого аккаунта программа сама заходит в настройки Dota 2 и выставляет:
- Все настройки графики → MIN / LOW
- Разрешение экрана → фиксированное (например 1024×768 или 800×600)
- Режим окна → оконный

Никаких ручных действий.

---

## 6. Система 50%

- Программа автоматически решает, какие аккаунты выигрывают, какие проигрывают.
- Баланс побед/поражений ≈ 50% / 50%.
- Никакого ручного выбора.
- Логирование: сколько побед/поражений у каждого аккаунта.

---

## 7. Механика GG

### Условия
- Настраиваемое время GG — пользователь может поставить любое (25, 30, 35, 40 минут и т.д.).
- GG работает только после заданного времени.

### Случайная задержка (анти-паттерн)
- GG не отправляется ровно в Х:00.
- Программа добавляет случайную задержку от 0 до 90 секунд.
- Пример: время 30 минут → GG уйдёт случайно между 30:00 и 31:30.

### Как отправляется
- Программа открывает чат игры (клавиша Enter).
- Печатает GG или GGWP.
- Нажимает Enter для отправки.
- НЕ нажимает Escape / Cancel / клик мимо чата.

> GG пишется в чат  
> При написании GG боты не нажимают кнопку отмены — только отправка через Enter

### Результат
- Команда, которая написала GG, автоматически проигрывает (как сдача в КС).

using System.Management;
using System.Security.Cryptography;
using System.Text;

namespace KeyGen
{
    class Program
    {
        static void Main()
        {
            // 1. Получаем HWID (железо)
            string hwid = GetHardwareId();
            // 2. Секретная соль (только ты знаешь)
            string salt = "DotaBot_SuperSecret_2026";
            // 3. Дата окончания (например, +30 дней)
            DateTime expiry = DateTime.UtcNow.AddDays(30);
            // 4. Генерируем ключ: "HWID|соль|дата"
            string raw = $"{hwid}|{salt}|{expiry:yyyy-MM-dd}";
            // 5. Шифруем ключ (AES)
            string encrypted = Encrypt(raw);
            // 6. Выводим готовый ключ
            Console.WriteLine($"Лицензионный ключ: {encrypted}");
            Console.WriteLine("Отправь этот ключ покупателю.");
        }

        static string GetHardwareId()
        {
            var sb = new StringBuilder();
            try
            {
                using var searcher = new ManagementObjectSearcher("SELECT ProcessorId FROM Win32_Processor");
                foreach (var obj in searcher.Get())
                    sb.Append(obj["ProcessorId"]);
            }
            catch { }

            try
            {
                using var searcher = new ManagementObjectSearcher("SELECT SerialNumber FROM Win32_DiskDrive WHERE Index=0");
                foreach (var obj in searcher.Get())
                    sb.Append(obj["SerialNumber"]);
            }
            catch { }

            using var sha = SHA256.Create();
            var hash = sha.ComputeHash(Encoding.UTF8.GetBytes(sb.ToString()));
            return Convert.ToBase64String(hash);
        }

        static string Encrypt(string plainText)
        {
            using var aes = Aes.Create();
            aes.Key = Encoding.UTF8.GetBytes("DotaBot_KeyGen_2026_SecureKey_32");
            aes.GenerateIV();
            var encryptor = aes.CreateEncryptor();
            var plainBytes = Encoding.UTF8.GetBytes(plainText);
            var encrypted = encryptor.TransformFinalBlock(plainBytes, 0, plainBytes.Length);
            var result = new byte[aes.IV.Length + encrypted.Length];
            Buffer.BlockCopy(aes.IV, 0, result, 0, aes.IV.Length);
            Buffer.BlockCopy(encrypted, 0, result, aes.IV.Length, encrypted.Length);
            return Convert.ToBase64String(result);
        }
    }
